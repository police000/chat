# DARKWIRE — термінальний чат

Повноцінний бекенд на Node.js (Express + WebSocket). Кімнати та повідомлення
зберігаються у файлах у папці `data/` на сервері — це вже не залежить від Claude
чи браузера, працює на будь-якому хостингу.

## Запуск локально

Потрібен Node.js 18+.

```bash
npm install
npm start
```

Відкрий http://localhost:3000 — і вже можна створювати кімнати. Щоб перевірити
чат із кимось іншим локально, відкрий два вікна браузера на цій адресі.

## Як виставити в інтернет, щоб заходили реальні люди

Найпростіший шлях — безкоштовний/дешевий хостинг з підтримкою Node.js та
WebSocket (обов'язково перевір, що хостинг підтримує WebSocket — не всі
безкоштовні статичні хостинги це вміють).

### Варіант 1: Render.com (найпростіше)
1. Заливаєш цю папку в репозиторій на GitHub.
2. На render.com → New → Web Service → підключаєш репозиторій.
3. Build command: `npm install`, Start command: `npm start`.
4. Після деплою отримаєш посилання типу `https://твій-проєкт.onrender.com` —
   можна одразу давати людям.
   ⚠️ На безкоштовному тарифі диск ефемерний: якщо сервіс перезапуститься
   (засне через неактивність), файли в `data/` можуть очиститись. Для
   постійного зберігання варто підключити платний persistent disk.

### Варіант 2: Railway.app
Аналогічно до Render — підключаєш репозиторій, Railway сам визначає
Node.js-проєкт і запускає `npm start`. Підтримує persistent volumes для
папки `data/`, якщо потрібне довготривале зберігання історії.

### Варіант 3: Fly.io (через Docker)
У проєкті вже є `Dockerfile`. Встановлюєш `flyctl`, у папці проєкту:
```bash
fly launch
fly deploy
```
Fly підтримує постійні volumes — можна змонтувати том у `/app/data`.

### Варіант 4: Власний VPS (DigitalOcean, Hetzner тощо) — повний контроль
1. Встанови Node.js 18+ на сервері.
2. Скопіюй туди цю папку, виконай `npm install`.
3. Запусти через `pm2` (щоб сервер не падав і піднімався після ребута):
   ```bash
   npm install -g pm2
   pm2 start server.js --name darkwire
   pm2 save
   pm2 startup
   ```
4. Постав nginx як реверс-проксі перед Node.js та випусти SSL-сертифікат
   через certbot (Let's Encrypt) — це дасть HTTPS і, відповідно, безпечний
   WSS для WebSocket. Приклад ділянки nginx-конфігу:
   ```nginx
   location / {
       proxy_pass http://127.0.0.1:3000;
       proxy_http_version 1.1;
       proxy_set_header Upgrade $http_upgrade;
       proxy_set_header Connection "upgrade";
       proxy_set_header Host $host;
   }
   ```
5. Прив'яжи свій домен (A-запис на IP сервера) — і сайт живе на твоєму домені.

## Важливо знати

- Історія повідомлень зберігається у простих JSON-файлах (`data/rooms.json`,
  `data/messages/<room-id>.json`) — цього достатньо для невеликого чату.
  Якщо навантаження виросте (сотні одночасних кімнат/користувачів), варто
  перейти на справжню базу даних (SQLite чи PostgreSQL) — можу допомогти
  переробити, якщо знадобиться.
- Є базовий захист від флуду (не більше 5 повідомлень / 3 секунди на
  з'єднання), обмеження довжини ніку (20 символів) і повідомлення (500
  символів).
- Чат **не** зашифрований end-to-end. Обов'язково використовуй HTTPS/WSS
  (SSL-сертифікат) на продакшн-домені, інакше трафік буде відкритим.
- Приватні кімнати (галочка при створенні) просто не показуються в
  публічному списку — вони не захищені паролем, доступ лише через знання ID.
  Якщо потрібен саме пароль на кімнату — скажи, додам.
